\begin{abstract}
We report numerical experiments for data-driven surrogate models of three PDEs: (1)~1D inviscid Burgers equation (finite-volume reference vs.\ MLP); (2)~2D linear wave equation (Fourier spectral reference vs.\ MLP); (3)~2D nonlinear rotating shallow water (Fourier spectral reference vs.\ ShrinkCNN). For each problem we generate training data from the reference solver, train a neural network, and compare reference and NN rollouts in time. Training curves and comparison figures are included. All configuration parameters are centralized in config files; PDE and training pipelines are unified under a single codebase.
\end{abstract}

\section{Background}\label{sec:background}

\subsection{Element learning}
The goal is to learn a \emph{local} map (one time step or a few) on a small spatial element (patch) from data produced by a high-fidelity solver. At inference time, the learned map is applied in a rollout over the domain, yielding a fast surrogate that can be compared to the reference solver in accuracy and runtime.

\subsection{Three test cases}
\begin{itemize}
  \item \textbf{1D Burgers (inviscid):} $u_t + u u_x = 0$ on $[0,L]$ with periodic BC. Reference: finite-volume Godunov flux. NN: MLP mapping a 1D patch (input length $2n_{\mathrm{st}}+n_{\mathrm{wd}}$) to interior ($n_{\mathrm{wd}}$ cells) over one step of $n_{\mathrm{jp}}$ time steps.
  \item \textbf{2D linear wave:} $u_{tt} = c^2(u_{xx}+u_{yy})$ with periodic BC. Reference: Fourier pseudo-spectral, exact modal time step. NN: MLP mapping two channels (displacement and velocity) on a 2D patch to the interior over one frame.
  \item \textbf{2D nonlinear shallow water:} Rotating shallow water in Fourier space with RK4. State $(h,q_x,q_y)$. Reference: spectral; NN: ShrinkCNN (patch in, patch out) with optional TV regularization in the loss.
\end{itemize}

\section{Conclusion}\label{sec:conclusion}
The pipelines for 1D Burgers, 2D linear wave, and 2D nonlinear shallow water run successfully with a unified workflow: \texttt{gen\_data.py}, \texttt{ml.train}, and \texttt{compare.py} are invoked with \texttt{--problem} to select the case. All configuration is in \texttt{config/}; PDE modules take parameters from config and contain no duplicate defaults. The reported figures show that the NN surrogates track the reference solutions and achieve noticeable speedups in the comparison runs.
